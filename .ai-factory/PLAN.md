## Goal

Добавить режим тренировки `Микс-трансформ`, в котором каждая буква текста остается на своем месте, но получает один из случайно выбранных типов движения:

- вращение по часовой стрелке;
- вращение против часовой стрелки;
- бесконечная трансформация между состояниями:
  - нормальное отображение;
  - переворачивание;
  - зеркальное отражение;
  - переворачивание и отражение одновременно.

Для трансформаций нужно избегать повторений соседних состояний, чтобы буква не выглядела зависшей в одном положении.

## Scope

Входит:

- новый режим `mix-transform` в состоянии приложения;
- новая radio-кнопка режима в интерфейсе;
- отображение режима в статусной плашке;
- рендеринг букв с индивидуально выбранным типом движения;
- CSS-анимации для вращения и трансформаций;
- учет текущих настроек скорости, уровня, паузы и показа оригинала;
- тесты для выбора эффектов и отсутствия повторов в последовательностях;
- обновление README, CHANGELOG, AGENTS и `.ai-factory/DESCRIPTION.md`.

Не входит:

- новые внешние библиотеки;
- Canvas/WebGL;
- изменение логики режимов перемешивания слов;
- отдельные настройки вероятностей для типов движения.

## Implementation Notes

- Сохранить существующую архитектуру: state -> renderer -> animation, без бизнес-логики в DOM.
- Добавить режим в `MODE_VALUES` как `mix-transform`.
- Для стабильности визуального результата использовать детерминированный выбор по индексу буквы и `shuffleSeed`/параметрам состояния, а не `Math.random()` на каждом кадре.
- Вращение можно переиспользовать из текущего режима, но добавить общий CSS-класс для смешанного режима.
- Для трансформаций лучше завести ограниченный набор CSS keyframes/классов с заранее заданными последовательностями без соседних одинаковых состояний.
- Пауза должна останавливать и вращение, и трансформации.
- `Показать оригинал` должен временно отключать микс-трансформ так же, как в остальных режимах.
- Демо-режим должен продолжать работать без отдельных изменений управления.
- Не использовать `innerHTML` для пользовательского текста.

### [x] Task 2: Add pure animation helpers for mixed effects

Deliverable: чистая логика выбора эффекта для каждой буквы вынесена в `src/animation.js`.

Files to change:

- `src/animation.js`
- optionally `tests/animation.test.js`

Expected behavior:

- Добавить helper, например `getMixTransformStyle(index, state)`.
- Helper возвращает:
  - тип эффекта: `rotate` или `transform`;

  - длительность и задержку анимации с учетом `level` и `speed`;
  - имя CSS-последовательности для `transform`.
- Выбор должен быть детерминированным для одного текста/состояния и достаточно распределенным по буквам.
- Для трансформаций использовать только последовательности без соседних повторов состояний.

Logging requirements:

- В helper логирование не добавлять, чтобы чистая логика оставалась тестируемой.
- Логировать агрегированную статистику в renderer после рендера: количество букв с вращением и трансформацией.
