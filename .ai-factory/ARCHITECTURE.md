# Архитектура: Layered Architecture

## Обзор

Для первой версии тренажера выбрана простая слоистая архитектура. Проект должен стартовать как статическое веб-приложение без сборщика, фреймворка и базы данных, поэтому важнее всего отделить чистую текстовую логику от DOM и CSS-анимаций.

Такой подход сохраняет низкую сложность MVP, но не превращает код в один большой скрипт. Позднее приложение можно будет перенести в Electron, Tauri или SPA-фреймворк, не переписывая алгоритмы преобразования текста.

## Обоснование решения

- **Тип проекта:** небольшое интерактивное клиентское приложение.
- **Технологический стек:** HTML, CSS, JavaScript.
- **Ключевой фактор:** основная сложность находится в преобразовании текста и управлении состоянием, а не в серверной логике.
- **Масштаб:** MVP без авторизации, базы данных и сетевых API.

## Структура папок

Рекомендуемая структура для первого этапа реализации:

```text
.
|-- index.html
|-- README.md
|-- AGENTS.md
|-- src/
|   |-- app.js                 # запуск приложения и связывание слоев
|   |-- state.js               # состояние приложения и обновление настроек
|   |-- text-transformer.js    # чистые функции преобразования текста
|   |-- renderer.js            # создание DOM для тренировочного текста
|   |-- animation.js           # параметры и управление анимацией
|   `-- controls.js            # обработчики формы и кнопок
|-- styles/
|   |-- base.css
|   |-- layout.css
|   `-- training.css
`-- tests/
    `-- text-transformer.test.js
```

Если проект останется совсем маленьким, допустимо начать с меньшего числа файлов, но `text-transformer.js` должен оставаться отделенным от DOM.

## Правила зависимостей

- `text-transformer.js` не импортирует DOM, CSS-классы и браузерные элементы.
- `renderer.js` может использовать результат преобразования текста, но не должен менять бизнес-правила преобразования.
- `controls.js` читает пользовательский ввод и вызывает обновление состояния.
- `animation.js` отвечает только за параметры движения, CSS-классы и паузу.
- `app.js` связывает модули и является точкой композиции.

Разрешенное направление зависимостей:

```text
controls -> state -> renderer -> text-transformer
app -> controls
app -> state
app -> renderer
renderer -> animation
```

Запрещено:

- вызывать DOM API из функций перемешивания текста;
- хранить состояние приложения в разрозненных глобальных переменных;
- смешивать расчет скорости, разбор текста и обработчики кнопок в одной функции;
- менять исходный текст при включении тренировочного режима.

## Коммуникация слоев

- Состояние хранится в одном объекте приложения.
- Пользовательские действия меняют состояние через явные функции.
- Рендеринг строится из текущего состояния.
- Чистые функции преобразования не имеют побочных эффектов.
- CSS-анимации управляются классами и CSS custom properties.

## Ключевые принципы

1. Сначала сохранить структуру текста, потом применять визуальные эффекты.
2. Алгоритмы преобразования должны тестироваться без браузера.
3. DOM-элементы должны пересоздаваться или обновляться предсказуемо при смене режима.
4. Пауза и временный показ оригинала не должны менять исходный текст.
5. Настройки скорости и уровня сложности должны иметь ограничения по диапазону.

## Примеры кода

### Чистое преобразование слова

```javascript
export function shuffleWordKeepingFirstLetter(word) {
  if (word.length <= 2) {
    return word;
  }

  const first = word[0];
  const rest = word.slice(1).split("");

  for (let index = rest.length - 1; index > 0; index -= 1) {
    const swapIndex = Math.floor(Math.random() * (index + 1));
    [rest[index], rest[swapIndex]] = [rest[swapIndex], rest[index]];
  }

  return first + rest.join("");
}
```

### Состояние без DOM внутри алгоритмов

```javascript
const initialState = {
  sourceText: "",
  mode: "rotating-letters",
  rotationType: "smooth",
  level: 1,
  speed: 1,
  isPaused: false,
  isOriginalVisible: false
};
```

### Управление CSS-анимацией через класс

```javascript
export function setPaused(rootElement, isPaused) {
  rootElement.classList.toggle("is-paused", isPaused);
}
```

```css
.is-paused .rotating-letter {
  animation-play-state: paused;
}
```

## Антипаттерны

- Не писать всю логику в `index.html`.
- Не использовать `innerHTML` с пользовательским текстом без экранирования.
- Не пересчитывать случайное перемешивание при каждом кадре анимации.
- Не терять пробелы и переносы при разбиении текста.
- Не добавлять фреймворк до появления реальной необходимости.
