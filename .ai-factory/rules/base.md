# Базовые правила проекта

> Автоматически подготовленные правила для текущей стадии проекта. Код приложения еще не создан, поэтому часть правил задает стартовые соглашения для первого MVP.

## Именование

- Файлы: использовать `kebab-case`, например `text-transformer.js`.
- Переменные и функции JavaScript: использовать `camelCase`.
- Классы JavaScript, если появятся: использовать `PascalCase`.
- CSS-классы: использовать понятные имена в `kebab-case`, например `training-preview`.
- Имена режимов хранить как стабильные строковые значения, например `rotating-letters` и `shuffled-words`.

## Структура модулей

- Не смешивать обработку DOM с алгоритмами преобразования текста.
- Текстовые преобразования держать в отдельных функциях, которые принимают строку и настройки, а возвращают данные для отображения.
- Состояние приложения хранить централизованно: исходный текст, активный режим, скорость, уровень, пауза, временный показ оригинала.
- Анимационные параметры должны вычисляться отдельно от разметки.

## Обработка ошибок

- Пустой текст должен показывать пустую область предпросмотра без ошибок.
- Короткие слова из 1-2 букв не нужно перемешивать.
- Пунктуация и пробелы не должны теряться при преобразованиях.
- Любые пользовательские настройки скорости и длительности нужно ограничивать разумными минимумами и максимумами.

## Логирование

- Для MVP не добавлять постоянное пользовательское логирование.
- Для отладки допустим `console.debug`, но перед готовой версией такие вызовы нужно убрать или закрыть флагом отладки.

## Тестирование

- При появлении тестов в первую очередь покрыть чистые функции:
  - разбор текста на токены;
  - перемешивание букв внутри слов;
  - сохранение пробелов, переносов и пунктуации;
  - расчет скорости по уровню сложности.

## UI-правила

- Первый экран должен быть рабочим интерфейсом, а не посадочной страницей.
- Кнопки управления должны быть доступны рядом с областью тренировки.
- Настройки скорости, уровня и режима должны быть видны без перехода на отдельную страницу.
- Текст не должен перекрывать элементы управления на мобильном и desktop-размерах.
