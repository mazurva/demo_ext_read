## Общая схема

Приложение работает как статическая страница. `index.html` подключает стили и ES-модули, а логика разделена по небольшим файлам в `src/`.

Поток данных:

1. Пользователь меняет текст или настройки.
2. `controls.js` обновляет состояние через `state.js`.
3. `app.js` запускает перерисовку.
4. `renderer.js` строит DOM для выбранного режима.
5. `animation.js` рассчитывает стили анимаций.
6. `training.css` применяет визуальные эффекты.

## Основные файлы

| Файл | Роль |
| --- | --- |
| `index.html` | Разметка приложения и подключение модулей. |
| `src/app.js` | Инициализация приложения и связывание модулей. |
| `src/state.js` | Единое состояние, валидация настроек и обновления. |
| `src/controls.js` | Обработчики формы, кнопок, демо-режима и клавиши `Esc`. |
| `src/renderer.js` | Рендер тренировочного текста для всех режимов. |
| `src/animation.js` | Расчет параметров вращения и `Микс-трансформ`. |
| `src/text-transformer.js` | Перемешивание букв внутри слов. |
| `src/logger.js` | Безопасное логирование событий без полного текста пользователя. |
| `styles/training.css` | Макет, состояния интерфейса и keyframes анимаций. |
| `tests/*.test.js` | Автоматические тесты бизнес-логики. |

## Принципы реализации

- текст рендерится через DOM API, без сборки HTML-строк из пользовательского ввода;
- полный пользовательский текст не пишется в логи;
- режимы перемешивания работают детерминированно от seed-значения;
- `Микс-трансформ` избегает повторяющихся соседних состояний в последовательностях;
- демо-режим не содержит интерактивных кнопок и закрывается по `Esc`.

## Расширение режимов

Для нового режима обычно нужно:

1. добавить значение режима в `MODE_VALUES` в `src/state.js`;
2. добавить радиокнопку в `index.html`;
3. добавить ветку рендера в `src/renderer.js`;
4. добавить расчет визуального поведения в `src/animation.js` или `src/text-transformer.js`;
5. обновить тесты и документацию.
